=== PATTERNS ===

- Gmail dedup via Message-ID: Store Message-IDs in sheet column F; skip if already processed; avoids is:unread dependency
- Per-user polling isolation: Each user gets independent Gmail+Sheets service from their own OAuth token; errors isolated
- Scope mismatch detection: In get_creds(), compare requested scopes vs stored token scopes; delete and force re-auth on mismatch
- AI prompt template escaping: Use {{ and }} in f-strings that contain JSON template literals to avoid KeyError
- External cron keepalive: Use cron-job.org to ping app every 15 min to prevent free-tier sleep
- AI + regex dual parser: Try AI first (NVIDIA/Gemini/Groq), fall back to 50+ regex patterns if API fails
- Multi-user token isolation: Store per-user OAuth tokens as credentials/token_<base64(email)>.json for independent Gmail sessions
- SVG pill labels: Always wrap text over arrows in <g><use href="#pill-COLOR"/><text/></g> to hide lines behind text
- SVG arrow color rule: Arrow matches DESTINATION box color, not source (Flask→Gmail = red, not blue)
- SVG alignment: All boxes in same layer share same y coordinate; equal gaps between columns (150px margins, 100px gaps)

### What It Does
Polls Gmail every 15 min for job application emails → AI/Regex parse → logs to Google Sheets → sends Telegram/Slack/WhatsApp alerts. $0 infrastructure.

### Architecture (Final)
- Flask app (webui.py) with OAuth + scheduler
- Gmail API fetch → Parser (AI + regex) → Google Sheets + Notifier
- Multi-user: each user connects own Gmail, tokens isolated as credentials/token_<base64(email)>.json
- 24/7 via cron-job.org pinging /cron/<secret> every 15 min

### 4 Approaches Tried
1. Flask v1 — full control but no UI, single-user
2. n8n — visual workflow but Android tracing-stop kills Node.js threads
3. Manus AI — autonomous but 300 credits/day insufficient for 672 polls/day
4. Flask v2 (final) — all learnings combined, PythonAnywhere deploy

### OAuth Scopes
- Gmail: https://www.googleapis.com/auth/gmail.readonly, https://www.googleapis.com/auth/gmail.modify
- Sheets: https://www.googleapis.com/auth/spreadsheets
- UserInfo: https://www.googleapis.com/auth/userinfo.email

### Google Sheet Schema (13 columns, A-M)
Company, Role, Date, Type (Offer/Rejection/Interview/Received), Summary, Message-ID, Email, Location, Salary, Next Step, Parser, Status, Timestamp

### AI Providers
- Primary: NVIDIA (meta/llama-3.1-70b-instruct) — migrated from Gemini due to 429 rate limits
- Fallback: Gemini, Groq
- Regex fallback: 50+ company patterns, 15+ role patterns, keyword type classifier
- Config: AI_PROVIDER=nvidia (default), GEMINI_API_KEY, GROQ_API_KEY, NVIDIA_API_KEY

### Notification Channels
1. Telegram Bot API (@GotJobAlert_bot) — DM format with company, role, date, type
2. Slack Webhook — POST to configured webhook URL
3. WhatsApp CallMeBot — unreliable (gateway sometimes doesn't send API key)
4. Pushover — API-based push notifications

### Deployment (PythonAnywhere Free)

- Venv: mkvirtualenv offertracker --python=python3.10
- cron-job.org: GET /cron/<secret> every 15 min prevents free-tier sleep
